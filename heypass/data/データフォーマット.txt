各ステータスは必須項目を除けば必要なものだけ書けば良い。
無記入のステータスはデフォルト値が適用される。

キーワードは半角英字、半角:で区切って値を書く。
値が文字列の場合は""で値を囲むこと。
複数指定可能な値は[]で囲み,で各値を区切る。 例：[0,1,2]

【グループ】
group：グループ名、文字列
prefixID：グループのID接頭辞
commonStat：共通ステータス(次のgroupまたはcommonStatが現れるまで有効)、値は各ステータスに従う

【パーツデータ】
id：数値 or 文字列、必須（グループ接頭辞が着いた状態で部位内重複不可）
name：パーツ名、文字列、必須
type：タイプ、文字列（陸:l、空:a、砲:c、補:s ※半角小英）、条件必須
size：サイズ、文字列（SS、S、M、L、LL ※半角大英）、条件必須
cost：コスト、数値
capa：キャパシティ、数値
hp：HP、数値
str：STR、数値
tec：TEC、数値
wlk：WLK、数値
fly：FLY、数値
tgh：TGH、数値
wbid：デフォルトWBのID、数値
slot：スロットの数、数値（0〜3）

auraStat:システム起動時のステータス、値は変更される各ステータス(変更されない値は書く必要は無い)
auraSystem:数値（0：無し、1:有り）

joint：{  各ジョイントの数
　hd：HDジョイントの数、BDパーツのデフォルト値１
　lg：LGジョイントの数、BDパーツのデフォルト値１
　bs：BSジョイントの数、BDパーツのデフォルト値１
　am：AMジョイントの数、BDパーツのデフォルト値２
　wp：WPジョイントの数、AMパーツのデフォルト値１
　hdac：HDACジョイントの数、HDパーツのデフォルト値１
　fcac：FCACジョイントの数、HDパーツのデフォルト値１
}

builtin：{
	lg：LGパーツ内蔵、数値（0：解除、1：内蔵）
	bs：BSパーツ内蔵、数値（0：解除、1：内蔵）
}

tunable：{
	mwp：MAIN武器チューン属性、数値（0：不可、1：可能）
	pwp：近接武器チューン属性、数値（0：不可、1：可能）
}

jointPart：ジョイントパーツ属性、値は0より上の数値
※この属性を設定した場合、ジョイント数の本来のデフォルト値が0、このパーツの部位ジョイント数が1に設定される。
  HDJの場合：ジョイントのデフォルト値が HDAC:0、FCAC:0、HD:1 に設定される。
  AMJの場合：ジョイントのデフォルト値が WP:0、AM:1 に設定される。

main：{  MAIN武器
　str：ダメージへのSTR依存度[%]、数値（0〜100）、デフォルト値 0
　tec：ダメージへのTEC依存度[%]、数値（0〜100）、デフォルト値 100（strに値が設定されている場合は 0）
　pwr：威力、数値、内蔵強化時の値を複数指定可、強化数分書く ※最後まで値が変わらない場合は1つで良い
　shl：弾数、数値、同上
　rng：射程、数値、同上
　spd：速度、数値、同上
　itv：間隔、数値、同上
}

sub：{  SUB武器
　str：ダメージへのSTR依存度[%]、数値（0〜100）、デフォルト値 0
　tec：ダメージへのTEC依存度[%]、数値（0〜100）、デフォルト値 100（strに値が設定されている場合は 0）
　pwr：威力、数値、内蔵強化時の値を複数指定可、強化数分書く ※最後まで値が変わらない場合は1つで良い
　shl：弾数、数値、同上
　rng：射程、数値、同上
　spd：速度、数値、同上
　itv：間隔、数値、同上
}

cartridge:{  カートリッジ、BDパーツ必須
　hpup：キャパシティアップ取得時のHPUP量、数値
　lv1：Lv1から取得可能なカートリッジ、文字列（カートリッジID (*数) @コスト ※*数はそのカートリッジが複数で無い場合は必要ない）
　lv6：Lv6以上で取得可能なカートリッジ、同上
　lv9：Lv9以上で取得可能なカートリッジ、同上
}

comment：コメント、文字列、複数指定可（改行 ※3行まで）

fixedParts：{  固定パーツ
　hd：ジョイントに固定されるパーツ、名前 or パーツデータを直接記入
　lg：同上
　bs：同上
　am：同上
　wp：同上
　hdac：同上
　fcac：同上
}

【fixedParts詳細】
各パーツの値は複数指定が可能。上から順に装着される。
fixed.json内に記入したパーツ名を指定することでリンクすることも可能。
fixedParts内のパーツデータのみnumキーワードが有効。num分だけパーツデータが複製される。
固定パーツのタイプ、サイズは指定が無い場合、親パーツの値を引き継ぐ。

【値の優先順位】
高　パーツステータス
│　commonStat
低　デフォルト値
